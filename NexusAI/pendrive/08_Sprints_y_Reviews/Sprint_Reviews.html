Seguridad por diseño

El browser nunca habla directo con el backend de IA. Todo pasa por Moodle PHP. Las claves de API nunca llegan al cliente.

⚡

Fallback graceful

Si el retrieval falla (cuota, Redis caído), el chat sigue sin contexto en lugar de devolver error 503.

📚

Indexación async

PDF → extracción → chunking 512 tokens / 64 overlap → embeddings → pgvector. Estado: pending → indexing → indexed

03 / 09
ADRs — Architecture Decision Records

Decisiones que tomamos

🗄️

Una sola base de datos para todo

En lugar de operar ChromaDB separado, extendimos PostgreSQL con pgvector. Un motor para datos relacionales y vectoriales. Queries SQL + vector en una operación. Backup con pg_dump estándar.

PostgreSQL 16pgvectorHNSW index
🔄

El proveedor de IA es intercambiable

La capa LLMProvider usa el SDK de OpenAI en modo compatibilidad. Cambiar de Gemini a GPT-4o-mini es solo cambiar variables de entorno, sin modificar código.

Gemini 2.5 Flash (dev)GPT-4o-mini (prod)
🔐

Seguridad por diseño — HMAC 3 capas

Bearer API key + firma HMAC-SHA256(timestamp ‖ nonce ‖ body) + anti-replay en Redis (ventana 5 min). El browser solo habla con Moodle. Sin CORS, sin JWT, sin OAuth.

HMAC-SHA256Redis noncesAnti-replay
🧩

Plugin local — no modifica Moodle

before_footer() inyecta el widget en todas las páginas sin que el docente configure nada. React compilado como bundle AMD único. Compatible Moodle 4.1–4.5 LTS.

local_nexusaiWebpack AMD4.1–4.5 LTS
